# Device Metrics Service

Сервис для сбора и анализа телеметрии устройств.
Реализован в виде REST API с использованием FastAPI.

---

## Функциональность

### Сбор данных

* Приём метрик от устройств:

```json
{
  "device_id": "string",
  "x": float,
  "y": float,
  "z": float
}
```

* Каждая запись сохраняется с временной меткой

---

### Аналитика

Для выбранного устройства или пользователя вычисляются:

* минимальное значение
* максимальное значение
* количество
* сумма
* медиана

Аналитика доступна:

* по устройству
* по пользователю (все устройства)
* по каждому устройству пользователя отдельно

---

### Пользователи и устройства

* Создание пользователей
* Привязка устройств к пользователям
* Получение агрегированной статистики

---

## Архитектура

* **FastAPI** - REST API
* **SQLAlchemy** - ORM
* **PostgreSQL** - база данных
* **Docker / docker-compose** - окружение

---

## Модель данных

### User

* id
* name (уникальный)

### Device

* id
* device_id (уникальный внешний идентификатор)
* user_id

### Measurement

* id
* device_id (FK)
* x, y, z (float)
* timestamp

## 📡 API

### Создание пользователя

```http
POST /users
```

---

### Создание устройства

```http
POST /devices
```

---

### Отправка метрик

```http
POST /measurements
```

---

### Аналитика по устройству

```http
GET /analytics?device_id=...&start=...&end=...
```

---

### Аналитика по пользователю

```http
GET /users/{user_id}/analytics
```

---

### Аналитика по устройствам пользователя

```http
GET /users/{user_id}/devices/analytics
```

---

## Обработка ошибок

| Ситуация                         | Код |
|----------------------------------|-----|
| Устройство не найдено            | 404 |
| Пользователь не найден           | 404 |
| Дубликат пользователя/устройства | 409 |
| Некорректный диапазон дат        | 400 |

---

## Ограничения и допущения

* Максимум 1000 записей на запрос (ограничение на уровне БД)
* Аналитика выполняется синхронно
* Имена пользователей считаются уникальными (упрощение)

---

## Производительность

* Индексы:

    * `device_id`
    * `timestamp`
* Предотвращена N+1 проблема через eager loading
* Ограничение выборки (`LIMIT 1000`)
